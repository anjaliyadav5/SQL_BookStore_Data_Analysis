# Book Store Data Analysis

## Overview

This project contains an SQL-based analysis of a bookstore database. The goal is to explore sales, customers, books, authors, and inventory to answer common business questions such as revenue trends, best‑selling books, and customer behavior.

The core of the project is the SQL script:

```
BookStore_Data_Analysis.sql
```

## What’s Inside

* Database schema assumptions (tables, keys, relationships)
* Analytical SQL queries
* Business‑oriented insights such as:

  * Top‑selling books and authors
  * Revenue by category or time period
  * Customer purchase patterns
  * Inventory and stock insights

## Requirements

To run this project, you’ll need:

* A relational database system (MySQL, PostgreSQL, or SQLite)
* An SQL client or command‑line tool

> ⚠️ Note: You may need to slightly adjust syntax depending on the database engine you use.

## How to Use

1. Create a new database in your SQL environment.
2. Import or create the required tables (based on your schema).
3. Open `BookStore_Data_Analysis.sql` in your SQL editor.
4. Run the queries section by section.
5. Review the output to extract insights.

## Example Questions Answered

* Which books generate the highest revenue?
* Who are the most valuable customers?
* Which genres perform best?
* Are there seasonal trends in sales?
